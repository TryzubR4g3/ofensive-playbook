# BloodHound & SharpHound

Used for mapping Active Directory domains as a graph to identify attack paths (like shortest path to Domain Admin, DCSync rights, etc).

## 1. SharpHound (Data Collector)

Run this from a compromised machine within the AD domain.

### Full Enumeration (Initial Pass)
```powershell
.\SharpHound.exe --CollectionMethods All --Domain $DOMAIN --ExcludeDCs
```
Used on: **adenumeration**

### Active Sessions Only (Faster, Periodic)
Useful to run periodically to catch users logging in and out.
```powershell
.\SharpHound.exe --CollectionMethods Session --Domain $DOMAIN --ExcludeDCs
```
Used on: **adenumeration**

*Note: This generates a timestamped `.zip` file that you need to transfer back to your attacker machine.*

## 2. BloodHound (GUI & Database)

Run this on your attacker machine.

### Start the Backend (Neo4j)
```bash
sudo neo4j console
```
*Default credentials: `neo4j` / `neo4j` (prompts for password change on first login).*

### Start the GUI
```bash
bloodhound --no-sandbox
```

Once open, simply drag and drop the `.zip` file generated by SharpHound into the BloodHound GUI to ingest the data.

## Useful Pre-Built Queries
- **Shortest Paths to Domain Admins**: The primary goal.
- **Find Principals with DCSync Rights**: Useful for creating Golden Tickets or dumping NTDS.
- **Find AS-REP Roastable Users**: Users that don't require Kerberos pre-authentication.
- **Find Kerberoastable Users**: Service accounts with SPNs.
